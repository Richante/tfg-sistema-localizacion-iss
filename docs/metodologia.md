# Metodología y Ciclo de Vida del Proyecto

Para el desarrollo de este TFG, se ha optado por un **Modelo de Ciclo de Vida en Espiral**, complementado con **Kanban** para la gestión operativa.

## 1. Justificación del Modelo en Espiral
Dada la naturaleza experimental del proyecto (predicción mediante IA y telemetría orbital), el modelo en espiral permite:
- **Gestión de Riesgos:** Identificar fallos en la precisión del modelo SGP4 antes de entrenar la Red Neuronal.
- **Desarrollo Iterativo:** Refinar la captura de datos antes de pasar a la visualización 3D.

## 2. Gestión de Tareas (Kanban)
Se utiliza un tablero Kanban para visualizar el flujo de trabajo:
- **Backlog:** Ideas y requisitos futuros.
- **To Do:** Tareas de la iteración actual.
- **In Progress:** Tarea activa (actualmente: Ingeniería de Datos).
- **Done:** Tareas finalizadas y validadas.

## 3. Control de Versiones
Uso de **Git y GitHub** siguiendo la filosofía de "Atomic Commits" para asegurar la trazabilidad de cada cambio en la lógica orbital.
